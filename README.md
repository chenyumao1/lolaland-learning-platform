# 🌟 LolaLand - 可爱的英语学习乐园 🌈

LolaLand是专为儿童设计的英语学习平台，采用可爱温馨的界面设计，为小朋友们提供快乐的英语学习体验。

## 🎯 平台特色

- 🌈 **可爱界面设计**：粉色系配色，适合儿童的视觉体验
- 🔐 **用户权限管理**：安全的登录系统，管理课程访问权限
- 📚 **四大课程体系**：全面覆盖英语学习各个方面
- 🎥 **视频教学**：支持多媒体教学内容
- 💖 **用户友好**：简单易用，专为儿童设计

## 📚 课程体系

### 🔤 Phonics
- **26个字母基础学习**
- 字母发音和书写练习
- 循序渐进的学习方式

### ⚡ Power up
- **Pre/G1/G2 综合能力提升**
- 分级教学，针对不同年龄段
- 全面提升英语综合能力

### 🚀 Journeys
- **GK/G1/G2 深度阅读理解**
- 培养阅读理解能力
- 文学欣赏和批判思维训练

### ✍️ Grammar & Writing
- **18个精品语法写作课程**
- 系统性的语法教学
- 多种文体的写作训练

## 👤 用户端使用指南

### 1. 登录系统
**示例账号（仅供测试）：**
- 用户名：`guest`
- 密码：`guest`
- 说明：体验账号，无课程权限，可查看预览内容

### 2. 登录界面特点
- 🎀 可爱的粉色界面设计
- 💫 浮动动画效果
- 🌈 用户名支持大小写输入
- ✨ 成功登录会有气球庆祝动画

### 3. 主界面导航
- **顶部**：LolaLand标题和欢迎信息
- **左侧边栏**：用户信息和课程状态
- **主区域**：四个课程标签页
- **底部**：版权信息

### 4. 课程学习流程

#### Phonics 课程
1. 点击 "🔤 Phonics" 标签页
2. 选择学习等级（Level 1-5）
3. 点击任意字母开始学习
4. 观看教学视频
5. 使用"返回"按钮继续学习其他字母

#### 其他课程
1. 点击相应的课程标签页
2. 选择适合的等级（Pre/G1/G2 或 GK/G1/G2）
3. 浏览课程特色和学习内容
4. 根据课程权限访问内容

### 5. 用户权限说明
- ✅ **已解锁课程**：可以正常学习所有内容
- 🔒 **未解锁课程**：只能查看预览内容
- 📞 **获取权限**：联系管理员开通课程

## 🔧 管理端操作指南

### 1. 用户管理

#### 添加新用户
编辑 `users_data.json` 文件，添加新用户：
```json
{
  "新用户名": {
    "password": "用户密码",
    "name": "显示姓名",
    "email": "邮箱地址",
    "purchased_courses": ["phonics", "power_up"],
    "purchase_date": "2024-06-21"
  }
}
```

#### 课程权限管理
在 `purchased_courses` 数组中添加或删除课程权限：
- `"phonics"` - Phonics课程
- `"power_up"` - Power up课程
- `"journeys"` - Journeys课程
- `"grammar_writing"` - Grammar & Writing课程

### 2. 内容管理

#### 视频文件管理
1. **目录结构**：
   ```
   videos/
   ├── a.mp4    # 字母A的视频
   ├── b.mp4    # 字母B的视频
   └── ...      # 其他字母的视频
   ```

2. **视频上传方式**：
   - **方式一**：直接将视频文件放入 `videos/` 目录
   - **方式二**：通过应用界面上传（当视频不存在时会显示上传界面）

3. **视频格式要求**：
   - 支持格式：MP4、MOV、AVI
   - 建议格式：MP4（兼容性最好）
   - 文件命名：`{字母小写}.mp4`

### 3. 部署管理

#### 本地部署
```bash
# 1. 克隆代码
git clone [repository-url]

# 2. 安装依赖
pip install streamlit

# 3. 运行应用
streamlit run app.py
```

#### 云端部署（Streamlit Cloud）
1. 将代码推送到GitHub仓库
2. 在 Streamlit Cloud 中连接仓库
3. 配置部署参数
4. 等待自动部署完成

### 4. 系统维护

#### 用户数据备份
定期备份 `users_data.json` 文件，避免数据丢失。

#### 视频内容更新
- 替换现有视频：直接覆盖 `videos/` 目录中的文件
- 添加新视频：按照命名规则添加新文件
- 删除视频：从 `videos/` 目录中删除对应文件

#### 日志监控
- 查看Streamlit运行日志
- 监控用户登录情况
- 检查错误报告
